# Plans

Use this root for implementation-facing plans derived from approved specs, issue needs, or roadmap decisions.

Plans should name the milestone, affected code areas, implementation steps, validation commands, and issue candidates. When work is ready for GitHub tracking, create issue mirrors under `docs/superpowers/issues`.

Recommended frontmatter:

```yaml
title: Short plan title
milestone: M1 - Issue Workflow Hardening
status: draft
source_spec: docs/superpowers/specs/example.md
```
