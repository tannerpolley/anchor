# M2 - Pull Request Workflow Parity

## Purpose

Bring pull request workflows to the same operational quality as the issue workflow. This milestone covers PR grouping, filtering, detail rendering, commits, review state, comments, branch metadata, and browser handoff where the GitHub web UI is still required.

## Success Criteria

- PR lists and details use the same repository-selection model as issues.
- Commits, source and target branches, review status, and comments are visible from the tool window.
- Browser openings are intentional handoffs rather than required for normal browsing.
- UI states remain IDE-theme-aware and usable across supported JetBrains IDEs.
- Verification includes targeted tests plus plugin validation when extension points change.
