# Anchor Superpowers Project

This directory is the durable project layer for the Anchor fork. It gives future agents the roadmap, tracker rules, and artifact paths needed to turn plugin extension ideas into planned, issue-backed implementation work.

## Canonical Artifacts

- `PROJECT_CONTEXT.md` is the setup source of truth.
- `milestones/` contains roadmap index pages.
- `specs/` contains loose product, architecture, and design specs.
- `plans/` contains implementation-facing plans.
- `issues/` contains GitHub issue mirrors.

Use the lifecycle `spec -> plan -> issue`. Keep canonical specs, plans, and issues in the flat roots above; milestone pages should link to them instead of owning nested copies.

## Workflow Skills

- `$superpowers-project:setup-project`
- `$superpowers-project:brainstorm-spec`
- `$superpowers-project:write-plan`
- `$superpowers-project:create-issues`
- `$superpowers-project:implement-plan`
- `$superpowers-project:resolve-issue`
- `$superpowers-project:align-project`
- `$superpowers-project:merge-changes`
- `$superpowers-project:audit-project`
