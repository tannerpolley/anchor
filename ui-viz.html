Revamp New Transaction for faster UX

by alph-a07 · opened 2 hours ago · 4 comments
The new transaction sheet currently takes ~1.2s to become interactive on mid-range devices.

This PR reduces it to ~300ms by:
• Pre-fetching account list at app launch
• Using lazy composables for category picker
• Caching recent payee suggestions locally

Closes #99
